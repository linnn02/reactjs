## lab 1.1 итоги
В этой лабораторной работе я реализовала одно и то же поведение (переключение подсветки абзаца по клику на кнопку) двумя способами. императивно через jQuery и декларативно через React.
В jQuery я напрямую работаю с DOM. Я нахожу элементы на странице и вручную описываю шаги изменения интерфейса. По клику на кнопку выполняется команда toggleClass, которая добавляет или убирает CSS класс highlight у абзаца. То есть я сама контролирую конкретные действия в DOM. Такой способ понятен для простых страниц, но при росте количества состояний становится сложнее поддерживать, потому что приходится следить за множеством ручных изменений, обработчиков и условий.
В React подход декларативный. Вместо прямого изменения DOM я храню состояние isHighlighted в useState. Нажатие на кнопку меняет состояние и React автоматически делает повторный рендер компонента и обновляет DOM так, чтобы интерфейс соответствовал текущим данным. Это соответствует идеям “Declarative UI structures” и “Data changes over time”. интерфейс описывается как результат состояния, а изменения данных со временем приводят к обновлению UI без ручных изменений.
Декларативный подход лучше масштабируется, если добавить больше элементов и логики, проще поддерживать код, потому что UI строится из состояния. Также React оптимизирует обновления через virtual DOM. он сравнивает старую и новую версии (diffing) и применяет только минимальные изменения в реальном DOM (patching), что улучшает производительность.
