## Lab 1.2 
React можно назвать "view layer", потому что он отвечает за отображение данных в интерфейсе. Компонент получает данные (например, состояние state), возвращает JSX разметку, и React показывает её пользователю. Это соответствует цепочке: state > component > UI. При этом React не навязывает полный набор инструментов для всего приложения (например, роутинг или работу с сервером), а фокусируется на построении UI.
В компоненте Counter состояние count хранится через useState, начальное значение равно 0. На экране отображается число и две кнопки: Increment и Decrement. Когда пользователь нажимает кнопку, вызывается setCount и состояние меняется. Изза изменения state React запускает повторный рендер (re render): компонент выполняется заново, создаётся новое описание UI, и React обновляет страницу
React использует virtual DOM для представление интерфейса в памяти. После изменения state React сравнивает (diffing) предыдущую и новую версии виртуального дерева и применяет к реальному DOM только минимальные изменения (patching). Поэтому обновления происходят эффективно, что важно для производительности.
