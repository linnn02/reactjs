## Lab 7.1 Code Splitting with Suspense
**Code splitting** это разбиение приложения на отдельные части (чанки?), которые загружаются по мере необходимости. Это ускоряет первую загрузку приложения
В этом проекте страницы Dashboard/Settings/Profile загружаются лениво через `React.lazy()`:
lazy() возвращает компонент, который подгружается динамически через import()
Когда компонент ещё не загружен, React приостанавливает рендер

Компонент `Suspense` нужен, чтобы показать `fallback` во время загрузки lazy-компонента. В качестве fallback используется `LoadingSpinner` со спиннером на CSS

## Lab 7.2 Error Boundary
**Error Boundary** это компонент, который ловит ошибки JavaScript в дереве дочерних компонентов и вместо падения всего приложения показывает запасной интерфейс 

В проекте ErrorBoundary:
реализован как class component
использует `getDerivedStateFromError()` чтобы переключиться в состояние ошибки
имеет кнопку **Try Again**, которая сбрасывает состояние `hasError` и позволяет повторить загрузку
