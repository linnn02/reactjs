## SNIPPET_01 Singleton Object
Объяснение: Singleton контролирует существование единственного экземпляра, поэтому работает на уровне объектов 

## SNIPPET_02 Observer Object
Объяснение: Observer связывает издателя и подписчиков через объекты-наблюдатели и уведомления, это объектный механизм взаимодействия

## SNIPPET_03 Facade Object
Объяснение: Facade предоставляет единый объект-обёртку над подсистемой, скрывая детали 

## SNIPPET_04 Factory Method Class
Объяснение: Factory Method в GoF считается Class-level, потому что выбор создаваемого объекта обычно завязан на интерфейс/наследование и шаблон создания на уровне класса (в таблице Ch.6 он отмечен как Class)

## SNIPPET_05 Decorator Object
Объяснение: Decorator оборачивает конкретный объект другим объектом и динамически добавляет поведение

## SNIPPET_06 Prototype Object
Объяснение: Prototype создаёт новые объекты через клонирование существующего экземпляра-прототипа

## SNIPPET_07 Chain of Responsibility Object
Объяснение: запрос передаётся по цепочке объектов-обработчиков, каждый объект может обработать или передать дальше

## SNIPPET_08 Command Object
Объяснение: команды это объекты, которые инкапсулируют действие, история хранит объектыкоманды

## SNIPPET_09 Flyweight Object
Объяснение: Flyweight переиспользует общий объект (intrinsic state) между множеством объектов

## SNIPPET_10 Mediator Object
Объяснение: Mediator объект-посредник, через который общаются остальные объекты
