## Что решает Singleton
Паттерн Singleton гарантирует, что в приложении существует только один экземпляр класса и к нему можно получить доступ из любого места

## Реализация в этой работе
- `singleton.js` класс `ConfigManager`, который хранит конфигурацию приложения
- Экземпляр хранится в статическом свойстве `ConfigManager.instance`.
- `getInstance()` возвращает один и тот же объект
- Даже `new ConfigManager()` не создаёт второй объект: конструктор возвращает уже созданный экземпляр

## Когда использовать
- менеджер конфигурации
- подключение к БД
- логгер
- кэш/хранилище общих настроек

## Минусы/ограничения
- глобальное состояние может усложнять тестирование
- при неправильном использовании превращается в global variables
- может скрывать зависимости между модулями
